  - Backfills MANAGER/STAFF users with `EMP0001`, `EMP0002`, ...

## Employee Code Format

- Format: `EMP0001`, `EMP0002`, `EMP0003`.
- Generated by backend only.
- Unique through `employees.employee_code`.
- CUSTOMER and ADMIN users do not receive an employee code by default.

## User / Employee Relationship

- `users` remains the login account table.
- `employees` is a 1-1 profile table for MANAGER/STAFF users.
- If a user is created or updated to MANAGER/STAFF, the backend creates an employee profile if missing.
- If a user is updated away from MANAGER/STAFF, the employee profile is kept and set `inactive`.

## StoreUser Relationship

- `store_users` remains unchanged and references `users.id` through `staff_id`.
- Store assignment API is not implemented in this phase.
- Future StoreUser writes should validate that the target user has an active employee profile.
